Sistem Bilgisi

Bu tablo GitHub Actions tarafından her 5 dakikada bir güncellenir.

• Yeşil renk AL emirlerini veya kar durumunu gösterir

• Kırmızı renk SAT emirlerini veya zarar durumunu gösterir

• Veriler otomatik olarak her 30 saniyede bir yenilenir

• Veri Akışı: Local API → GitHub Actions → GitHub Pages JSON → Bu Sayfa

• Çevrimdışı durumda cache verileri gösterilir

Nasıl Çalışır?

1. GitHub Actions her 5 dakikada bir çalışır

2. Local API'nizden (192.168.1.3:3000) veri çeker

3. JSON formatında GitHub Pages'e kaydeder

4. Bu sayfa HTTPS üzerinden JSON'u okur ve tabloda gösterir

5. JavaScript her 30 saniyede bir verileri yeniler

6. Cache mekanizması ile çevrimdışı çalışabilir

Önemli Notlar

• GitHub Actions workflow'unun etkin olduğundan emin olun

• Local API'nizin 192.168.1.3:3000 adresinde çalıştığından emin olun

• GitHub Pages ayarlarında "Enforce HTTPS" aktif olmalı

• JSON dosyaları public klasöründe olmalı

• Tarayıcı doğrudan local API'ye erişemez, sadece GitHub Pages üzerinden veri alır

GitHub Actions Workflow

name: Fetch Data from Local API
on:
  schedule:
    - cron: '*/5 * * * *'  # Her 5 dakikada bir
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  fetch-data:
    runs-on: self-hosted
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
      
      - name: Fetch API Data
        run: |
          curl -s "http://192.168.1.3:3000/api/emirler" > public/data.json
          echo '{ "success": true, "lastFetch": "'$(date -u +"%Y-%m-%dT%H:%M:%SZ")'", "source": "http://192.168.1.3:3000", "emirler": '$(cat public/data.json)' }' > public/api-data.json
      
      - name: Commit and Push
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git add public/
          git commit -m "Update API data" || echo "No changes"
          git push

Bu workflow repository'nin .github/workflows dizininde olmalı.

İnternet bağlantısı kesildi. Çevrimdışı modda çalışıyorsunuz.
